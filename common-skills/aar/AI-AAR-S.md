# AI After Action Report: Small Edition (AI-AAR-S)

AI-AAR-S is the **minimum viable handoff** between jobs. Timebox: **60–180 seconds**. Target length: **6–12 bullets**. If you need paragraphs, upgrade to AI-AAR-M.

## 1) Goal (1 line)

What was the job trying to achieve?

* Format: *Verb + deliverable + target*
* Include the “definition of done” in 5–10 words.

## 2) Status (1 line)

Where are we now?

* **DONE / PARTIAL / BLOCKED**
* If partial/blocked: one phrase why.

## 3) Output Links (1–3 bullets)

Where is the work?

* Pointers only (paths/URLs/IDs), no long paste.

## 4) Key Learnings (2–4 bullets)

What should Job 2 know immediately?

* Prefer “surprising constraints,” “gotchas,” “facts discovered.”
* Tag: **Confirmed / Suspected / Validate**

## 5) Next Step (1–2 bullets)

What should happen first in Job 2?

* Put the very next command/action here.
* If there’s a known decision fork, name it.

## 6) Risks / Checks (1–2 bullets)

What could go wrong silently, and how to detect it?

* One verification step is ideal.
* Note any sensitive-data boundaries.
